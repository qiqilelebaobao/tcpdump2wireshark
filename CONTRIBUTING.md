# Contributing to tcpdump2wireshark

## 贡献指南

Thank you for your interest in contributing to tcpdump2wireshark! / 感谢您对 tcpdump2wireshark 的贡献！

## How to Contribute / 如何贡献

### Reporting Bugs / 报告错误

If you find a bug, please open an issue with:
- A clear and descriptive title
- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Your environment (OS, shell version, tcpdump version, Wireshark version)

如果您发现错误，请提交 issue 并包含：
- 清晰的标题
- 重现问题的步骤
- 期望的行为
- 实际的行为
- 您的环境信息（操作系统、shell版本、tcpdump版本、Wireshark版本）

### Suggesting Enhancements / 建议改进

Enhancement suggestions are welcome! Please open an issue with:
- A clear and descriptive title
- A detailed description of the proposed enhancement
- Any relevant examples or use cases

欢迎提出改进建议！请提交 issue 并包含：
- 清晰的标题
- 详细的改进描述
- 相关示例或使用场景

### Pull Requests / 提交拉取请求

1. Fork the repository / 复刻仓库
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes / 进行修改
4. Test your changes thoroughly / 彻底测试您的修改
5. Commit your changes with a clear message / 使用清晰的消息提交更改
6. Push to your fork / 推送到您的复刻仓库
7. Open a Pull Request / 提交拉取请求

#### Pull Request Guidelines / 拉取请求指南

- Keep changes focused and atomic / 保持更改集中且原子化
- Update documentation if needed / 如需要请更新文档
- Follow the existing code style / 遵循现有代码风格
- Add comments for complex logic / 为复杂逻辑添加注释
- Test on both Linux and macOS if possible / 如可能，在 Linux 和 macOS 上测试

## Code Style / 代码风格

- Use descriptive variable and function names / 使用描述性的变量和函数名
- Add comments in Chinese and English where helpful / 在有帮助的地方添加中英文注释
- Follow shell scripting best practices / 遵循 shell 脚本最佳实践
- Use functions to organize code / 使用函数组织代码

## Questions? / 有疑问？

Feel free to open an issue for any questions or discussions!

欢迎提交 issue 进行任何问题讨论！
